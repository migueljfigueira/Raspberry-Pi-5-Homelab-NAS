# Nextcloud AIO Backup to External Drive (Borg)

## Overview

This guide shows how to configure and run a proper backup of Nextcloud AIO using an external drive.



## 1. Verify Backup Directory

```bash
ls -ld /mnt/backup
df -h /mnt/backup
```

Ensure the directory exists.



## 2. Detect External Drive

```bash
lsblk
```

Identify the external disk (e.g. `/dev/sda2`).



## 3. Mount External Drive

```bash
sudo mount /dev/sda2 /mnt/backup
df -h /mnt/backup
```

Confirm it is mounted to `/mnt/backup`.



## 4. Stop Nextcloud (Apache)

```bash
docker stop nextcloud-aio-apache
```

Required to ensure consistent backup.



## 5. Fix Old Backup Configuration (if needed)

Check config:

```bash
sudo docker exec nextcloud-aio-mastercontainer ls -l /mnt/docker-aio-config/data/
```

If `borg.config` exists, remove it:

```bash
sudo docker exec nextcloud-aio-mastercontainer rm /mnt/docker-aio-config/data/borg.config
```



## 6. Configure Backup Location

Open AIO interface:

```
http://<PI-IP>:8080
```

Set backup path:

```
/mnt/backup
```

Click **Submit backup location**



## 7. Run Backup

Start backup from AIO interface.



## 8. Verify Backup

```bash
ls -lah /mnt/backup
```

Expected:

```
borg/
```

Optional:

```bash
ls -lah /mnt/backup/borg
```



## Notes

* Backups are stored using Borg
* Location: `/mnt/backup/borg`
* Always use external storage (not same disk as OS)
* Stop Apache before backup to avoid data inconsistency
