# Nextcloud AIO Installation (Raspberry Pi + Docker + Tailscale)

This guide documents the installation of **Nextcloud All-in-One (AIO)** on a **Raspberry Pi 5** using **Docker**, **Portainer**, and **Tailscale**.
The setup works behind **CGNAT** and avoids router port-forwarding.

---

# Architecture

```
Client (Phone / PC)
        │
     Tailscale
        │
https://cloudserver.tailXXXX.ts.net
        │
tailscale serve
        │
127.0.0.1:11000
        │
Nextcloud AIO (Docker)
        │
Raspberry Pi
```

Key properties:

* No public port forwarding
* Works behind CGNAT
* Encrypted via Tailscale HTTPS
* Accessible only inside the tailnet

---

# 1. Deploy Nextcloud AIO Stack (Portainer)

Create a new stack called **nextcloud**.

```yaml
services:
  nextcloud-aio-mastercontainer:
    image: nextcloud/all-in-one:latest
    init: true
    restart: always
    container_name: nextcloud-aio-mastercontainer
    ports:
      - 8080:8080
    volumes:
      - nextcloud_aio_mastercontainer:/mnt/docker-aio-config
      - /var/run/docker.sock:/var/run/docker.sock:ro
    environment:
      APACHE_PORT: 11000
      APACHE_IP_BINDING: 127.0.0.1

volumes:
  nextcloud_aio_mastercontainer:
    name: nextcloud_aio_mastercontainer
```

Deploy the stack.

---

# 2. Configure Tailscale Reverse Proxy

Enable HTTPS proxy to the Nextcloud Apache container.

```bash
sudo tailscale serve --https=443 http://127.0.0.1:11000
```

Verify:

```bash
tailscale serve status
```

Expected output:

```
https://cloudserver.tailXXXX.ts.net
|-- / proxy http://127.0.0.1:11000
```

---

# 3. Open Nextcloud AIO Setup

Open:

```
https://<tailscale-ip>:8080
```

Example:

```
https://100.92.143.101:8080
```

Accept the browser warning (self-signed cert).

---

# 4. Submit Domain

Enter the **Tailscale HTTPS domain**:

```
cloudserver.tailXXXX.ts.net
```

Do **not** use:

* public domains
* IP addresses
* localhost

This avoids Let's Encrypt validation issues behind CGNAT.

---

# 5. Start Nextcloud Containers

Click:

```
Start containers
```

AIO will create:

```
nextcloud-aio-apache
nextcloud-aio-nextcloud
nextcloud-aio-database
nextcloud-aio-redis
nextcloud-aio-collabora
nextcloud-aio-imaginary
nextcloud-aio-notify-push
```

Wait until containers show **healthy**.

---

# 6. Access Nextcloud

Open:

```
https://cloudserver.tailXXXX.ts.net
```

Create the admin account and log in.

---

# Notes

* Default storage uses Docker volumes:

```
/var/lib/docker/volumes/
```

* Since the OS is on NVMe, the full disk capacity is available to Docker volumes.

---

# Useful Commands

Check containers:

```bash
docker ps
```

Reset a user password:

```bash
docker exec -it nextcloud-aio-nextcloud php occ user:resetpassword USERNAME
```

Check Tailscale proxy:

```bash
tailscale serve status
```

# Result

A fully working **private cloud server** with:

* secure access via Tailscale
* no router configuration
* automatic TLS
* isolated Docker environment
