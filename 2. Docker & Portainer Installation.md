# Docker & Portainer Installation (Raspberry Pi Homelab)

This guide documents the basic installation of **Docker Engine**, **Docker Compose**, and **Portainer** on a Raspberry Pi server used for a homelab environment.



## 1. Verify System Architecture

Check the OS architecture (required for Docker packages):

```bash
dpkg --print-architecture
```

Expected output:

```
arm64
```



## 2. Update System Packages

```bash
sudo apt update
```

Refreshes the package index from configured repositories.



## 3. Install Required Dependencies

```bash
sudo apt install ca-certificates curl gnupg
```

These packages allow secure package downloads and repository verification.



## 4. Add Docker GPG Key

Create the key directory:

```bash
sudo install -m 0755 -d /etc/apt/keyrings
```

Download Docker’s official signing key:

```bash
curl -fsSL https://download.docker.com/linux/debian/gpg \
| sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
```



## 5. Add Docker Repository

```bash
echo "deb [arch=arm64 signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/debian bookworm stable" \
| sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

Update package lists again:

```bash
sudo apt update
```



## 6. Install Docker Engine

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

Verify Docker is running:

```bash
sudo systemctl status docker
```



## 7. Test Docker Installation

Run the test container:

```bash
sudo docker run hello-world
```

Expected output:

```
Hello from Docker!
This message shows that your installation appears to be working correctly.
```



## 8. Allow Docker Without sudo

Add the user to the Docker group:

```bash
sudo usermod -aG docker $USER
```

Reload group membership:

```bash
newgrp docker
```

Verify:

```bash
docker ps
```



## 9. Create Docker Workspace

```bash
mkdir -p ~/docker
```

Example structure for homelab services:

```
~/docker
 ├── compose
 ├── nextcloud
 ├── databases
 └── monitoring
```



## 10. Install Portainer

Create persistent storage:

```bash
docker volume create portainer_data
```

Run Portainer container:

```bash
docker run -d \
-p 9443:9443 \
--name portainer \
--restart=always \
-v /var/run/docker.sock:/var/run/docker.sock \
-v portainer_data:/data \
portainer/portainer-ce:latest
```



## 11. Access Portainer

Open the web interface:

```
https://SERVER-IP:9443
```

Example (local network):

```
https://192.168.0.102:9443
```

Example (Tailscale VPN):

```
https://100.x.x.x:9443
```

Create the admin account and connect to the **local Docker environment**.



## Result

The server now runs:

* Docker Engine
* Docker Compose
* Portainer container management UI

This provides the foundation for running homelab services such as:

* Nextcloud
* Databases (MariaDB / PostgreSQL)
* Reverse proxies
* Monitoring tools
