# Netdata Monitoring Setup (Raspberry Pi Homelab)

## Overview

Installed and configured **Netdata** for real-time monitoring of a Raspberry Pi server running Docker (Nextcloud stack).



## 1. Install Netdata (Docker)

```bash
docker run -d \
  --name=netdata \
  --hostname cloudserver \
  --pid host \
  --network host \
  --cap-add SYS_PTRACE \
  --cap-add SYS_ADMIN \
  --security-opt apparmor=unconfined \
  -v /:/host/root:ro,rslave \
  -v /proc:/host/proc:ro \
  -v /sys:/host/sys:ro \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  -v netdataconfig:/etc/netdata \
  -v netdatalib:/var/lib/netdata \
  -v netdatacache:/var/cache/netdata \
  netdata/netdata
```

Access dashboard:

```
http://<tailscale-ip>:19999
```



## 2. Dashboard Customization

Focus on key metrics:

### System

* CPU usage
* Load average
* RAM usage
* Temperature

### Storage

* Disk space
* Disk physical I/O

### Network

* Traffic (in/out)

### Optional

* Container state (Docker)



## 3. Create Local Alerts (No Cloud)

Enter container:

```bash
docker exec -it netdata bash
```

Go to alerts folder:

```bash
cd /etc/netdata/health.d
```

Create custom alert file:

```bash
vi custom.conf
```



## 4. CPU Temperature Alert

```conf
alarm: cpu_temperature_high
on: sensors.temperature_cpu_thermal-virtual-0_temp1_input
lookup: average -30s unaligned
units: Celsius
every: 10s
warn: $this > 75
crit: $this > 80
info: CPU temperature is too high
```

Restart Netdata:

```bash
exit
docker restart netdata
```



## 5. Verify Alert

```bash
curl -s http://localhost:19999/api/v1/config?action=tree | grep cpu_temperature_high
```

Check UI:

* Netdata → 🔔 Alerts



## Notes

* Netdata Cloud not required (using local alerts)
* Alerts configured via `/etc/netdata/health.d`
* Use real chart IDs (not context names)
* Monitoring covers system, disks, and Docker containers

* Real-time monitoring dashboard
